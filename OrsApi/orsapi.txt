    ## Introduction
    Welcome to the ORS APIv2 reference. This guide explains the ORS application programming interface (API). It describes various API operations, requests, response structures and errors.

    The API is REST-like with data exchanged in JSON format.

    Some useful links:
    - [ORS API pricing and licence info](https://orstravel.com/api/)
    - [Swagger doc](https://api.ors.si/docs/swagger)
    - [API v2 doc](https://api.ors.si/docs/v2)
    - [(old) API v2 doc](https://api.ors.si/docs/old-docs) for reference only, it will be removed by the end of 2025.

    ## Content Types
    In ORS there are currently 3 content types. This type must be included in search requests.

    Depending on a content type different fields are applicable in a search query and search results may have different fields present in its results.

    These content types are:
    - **pauschal**: Includes accommodation offers with flight transport included by a tour operator.
    - **hotel**: Includes only accommodation offers, with customer requiring to organize their own transport.
    - **trips**: Organized (and usually guided) roundtrips performed by tour operators.


    ## Recommended development flow
    If you're new to ORS API, then we suggest you follow this steps for a successfull integration:

    *Note: you can use `Quick Search` to create autocomplete input for selecting product or location*

    - step 1 (search):
        - display regions and destinations using `Search Regions` operation,
        - after selected region/destination display products for that region using `Search Products` request,
        - when product is selected, display possible dates (offers) using `Search Dates` request,
    - step 2 (product information):
       in order to display product description and pictures, you can call `Get product information` request
    - step 3 (availability):
       before you continue to Booking operations, you should first send `Availability Check` request to see if date is still available for reservation and what is the total price for all passengers.
    - step 4 (booking):
        - for wire transfers we recommend calling `Booking Registration` request, since this operation will not imediatelly confirm reservation, but it will allow you to manually confirm reservation once you receive payment,
        - for direct payments you can call `Booking` request, which will also create confirmed reservation on Touroperator side.

    ## Special request examples

    This are some of the common special request examples.

    ### Search for cruise offers

    You can find cruise offers in ORS under content type `trips`, by adding search parameter `Subtype:cruise`.

    ~~~json
    {
      "StartDate": "2025-09-23",
      "EndDate": "2025-11-30",
      "MinDuration": 2,
      "MaxDuration": 25,
      "AdultCount": 2,
      "Subtype": "cruise"
    }
    ~~~

    ### Search for SKI offers

    SKI offers can be found under `hotel` content type, by adding search parameter `ProductFacts:["ski"]`.

    ~~~json
    {
      "StartDate": "2025-09-23",
      "EndDate": "2025-11-30",
      "MinDuration": 7,
      "MaxDuration": 7,
      "AdultCount": 2,
      "ProductFacts": ["ski"]
    }
    ~~~

    The same can be acchived by using `Filter`:

    ~~~json
    {
      "StartDate": "2025-09-23",
      "EndDate": "2025-11-30",
      "MinDuration": 7,
      "MaxDuration": 7,
      "AdultCount": 2,
      "Filter": {
        "ProductFacts": ["ski"]
      }
    }
    ~~~

    ### Search for offers with included Skipass

    Offers that are tagged with `SKIPASS` tag have included Skipass in the price. So add special tag `Tags:["SKIPASS"]` in request.

    ~~~json
    {
      "StartDate": "2025-09-23",
      "EndDate": "2025-11-30",
      "MinDuration": 7,
      "MaxDuration": 7,
      "AdultCount": 2,
      "Tags": {
        "SKIPASS"
      }
    }
    ~~~
